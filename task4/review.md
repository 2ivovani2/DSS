# Лабораторная работа №4  
## Моделирование угроз безопасности информации

### Цель работы
Целью работы является построение модели угроз для разработанного в предыдущей лабораторной работе приложения с использованием диаграммы потоков данных (DFD) и методологии STRIDE, а также проведение оценки рисков выявленных угроз по методике DREAD.

---

# 1. Описание системы

В рамках предыдущего задания было разработано консольное приложение на языке Python, которое взаимодействует с базой данных PostgreSQL. Приложение позволяет пользователю выполнять операции с данными через командную строку.

Основные компоненты системы:

- Пользователь (User)
- Консольное приложение (CLI Application)
- База данных PostgreSQL
- JSON-файл с данными

Пользователь вводит команды через консольное приложение. Приложение обрабатывает введённые данные и выполняет соответствующие SQL-запросы к базе данных. Результаты работы возвращаются пользователю через интерфейс командной строки.

---

# 2. Диаграмма потоков данных (DFD)

Диаграмма потоков данных включает следующие элементы:

**Внешняя сущность**
- User

**Процессы**
- CLI Application

**Хранилища данных**
- PostgreSQL Database
- JSON File

**Потоки данных**
- Пользователь вводит команды в CLI приложение
- JSON файл передает данные приложению
- Приложение выполняет SQL-запросы к базе данных
- База данных возвращает результаты запросов
- Приложение выводит результаты пользователю

*(В отчёт необходимо вставить диаграмму, созданную в OWASP Threat Dragon.)*

# 3. Локализация угроз по методологии STRIDE

| Компонент системы | Угроза | Тип STRIDE |
|---|---|---|
| User → CLI Application | Подмена пользователя при вводе команд | Spoofing |
| User → CLI Application | Отрицание пользователем выполнения действий | Repudiation |
| CLI Application | Выполнение вредоносных команд через пользовательский ввод | Tampering |
| CLI Application | Перегрузка приложения большим количеством запросов | Denial of Service |
| CLI Application | Отсутствие журналирования действий | Repudiation |
| CLI Application → PostgreSQL | SQL-инъекция через параметры запроса | Tampering |
| PostgreSQL DB | Несанкционированный доступ к данным | Information Disclosure |
| PostgreSQL DB | Изменение или удаление данных в БД | Tampering |
| JSON File | Подмена данных в JSON файле | Tampering |
| JSON File | Чтение конфиденциальных данных | Information Disclosure |
